01 / Theme

Fallback contract

Use this style only after a quick source-style search fails. It is intentionally generic, but not bland.

Paper first

The base is ivory and paper with visible borders. It reads as a designed document rather than an app shell.

ivory paper line

Plum accent

The site-inspired orange/clay accent is replaced by light plum for section rules, active tabs, links, and highlights.

plum plum-dark plum-soft

Controls wrap and select

Jump links, chips, tabs, legends, and button rows wrap. Selectable controls update state; only code panes, diffs, tables, and large diagrams may scroll internally.

document.addEventListener("click", (event) => {
  const tab = event.target.closest("[data-tab]");
  if (!tab) return;

  const group = tab.closest("[data-code-tabs]");
  const selected = tab.dataset.tab;

  group.querySelectorAll("[data-tab]").forEach((button) => {
    button.classList.toggle("is-active", button === tab);
    button.setAttribute("aria-selected", String(button === tab));
  });

  group.querySelectorAll("[data-panel]").forEach((panel) => {
    panel.classList.toggle("is-active", panel.dataset.panel === selected);
  });
});

04 / QA

Validation notes

This artifact is built to make the updated skill requirements directly testable.

Expected checks: nav and chip controls wrap, selected nav/chip state updates, no page-level horizontal overflow on mobile, code tabs switch panels, copy buttons read from the active snippet or target block, and the diagram uses routed non-crossing edges.

Source inspiration: the referenced HTML-effectiveness examples use editorial typography, ivory/paper surfaces, pill navigation, collapsible sections, tabbed code samples, and diff-like review blocks. This fallback preserves that spirit while replacing the orange/clay accent with light plum.
